# Pending validations

This directory contains scripts and notes for validations that should be performed but were not feasible in the original development environment.

## Why these are pending

Most of the core Alaniz Cipher v4r3 development was done in a sandboxed Python environment with ~2 GB RAM and process time limits. Some validations that are straightforward elsewhere (SageMath workstation, Magma, machines with more RAM) were left as TODOs.

These are not blockers for the scheme being publishable or analyzable — the components were each verified — but they would significantly strengthen the empirical foundation.

## What needs to be done, in priority order

### 1. `sage_d12_complete.sage` — Verify=True at d=12 PQ-128

**The single most important validation.** Completes the d=12 pipeline that was paused at the F4 RREF step.

```bash
sage sage_d12_complete.sage
```

**Expected**: `VERIFY = True` after ~2-5 minutes of compute.

**Inputs**: `experiments/data/d12_*.pkl` files (pickled by the Python development).

**What it does**:
1. Loads the persisted state from the Python session.
2. Identifies the correct σ⁻¹ candidate combination consistent with the original message.
3. Builds the Macaulay matrix at degree D=5 over F_(2⁶¹-1) (size 4368×6188).
4. Computes RREF using SageMath's native `Matrix.rref()`.
5. Extracts the recovered message from the unique kernel vector.
6. Verifies that the recovered message matches the original.

**What it confirms**: the recommended parameters tetra d=12, e=17, p=2⁶¹-1 yield a functional encryption scheme end-to-end. Combined with the security analysis, this validates the scheme at PQ-128 parameters.

### 2. Empirical D_reg at d ≥ 4 for the public system

**Goal**: extend the gap regression validation to d ≥ 4 to bridge the extrapolation gap to d=12.

**Method**: replicate the methodology of `experiments/reproduce_attack_public.py` (which works at d=2,3 in our environment) at d=4, 5, 6, 8.

**Software**: SageMath or Magma. Magma's `GroebnerBasis` function or SageMath's `Sequence(...).groebner_basis()` are appropriate.

**Compute requirements**:
- d=4, e=7 (smallest valid): Macaulay matrix ~30,000 × 66,000. ~16 GB RAM.
- d=5, e=11: ~150,000 × 350,000. ~64 GB RAM.
- d=6, e=17: ~500,000 × 1.5M. Requires server hardware.

**Estimated effort**: 1-2 weeks of compute, 1-2 weeks of analysis.

### 3. Independent F4/F5 attack against d=12 ciphertext

**Goal**: simulate an adversary with only public key + ciphertext, attempting to recover the message via direct F4 or F5 on the public polynomial system. Validate that the attack cost matches the Bardet-Faugère prediction.

**Method**: use Magma's optimized F4 implementation (the best publicly available) and time the attack at small `e` to verify scaling.

**Estimated effort**: 1 month, requires Magma license.

### 4. Side-channel statistical study at scale

**Goal**: confirm or refute the marginal observation that ρ(v_0, v_3) = -0.38 in our N=30 trial.

**Method**: N ≥ 1000 trials with multiple keys, rigorous χ² and mutual information analysis on σ⁻¹ root count distributions, decryption timings, error counts.

**Software**: any reasonable statistics environment after implementing an optimized C decryptor.

**Estimated effort**: 1 week of compute on optimized implementation.

### 5. Constant-time σ⁻¹ implementation

**Goal**: eliminate any potential timing side-channel from σ⁻¹ root count variation.

**Method**:
- Always enumerate all roots (no early termination).
- Pad enumeration to a maximum count.
- Randomize the order of combo trials.

**Software**: C or Rust implementation.

**Estimated effort**: 2-4 weeks within an optimized implementation project (which is a separate effort, see `docs/ROADMAP.md` priority 2.1).

## Reproducibility notes

All persisted state files in `experiments/data/` were generated by Python 3.12 with `numpy>=1.24` and `galois>=0.3.0`. They use standard pickle format and are platform-independent.

The state files are:
- `d12_state.pkl`: Public/private key, plaintext, ciphertext, nonce, field structure constants.
- `d12_structure.pkl`: Cubic system structure (independent of σ⁻¹ choice).
- `d12_cands_v0.pkl` ... `d12_cands_v3.pkl`: σ⁻¹ candidates at each vertex.
- `pq128_results.jsonl`: Results of 10-seed d=6 e2e verification.
- `sidechan_state.jsonl`: Results of 30-trial side-channel study.
